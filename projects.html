Networking

Home Network Segmentation and Wi-Fi Mesh Setup

Fiber network, switch topology, mesh placement, and device separation.

I planned a home setup that improved coverage while separating devices by purpose, making the network easier to manage and troubleshoot. This is a growing project that will probably be redone when I move. I had it setup at a family members house to spread wifi over two houses. The family members live next to eachother and only wanted to pay for one internet plan (makes sense, I would too). The only thing I wish we did different was to run an ethernet backhaul from one house to another, but that was going to be too much work and they didn't want that. The plan for my next house will be to expand a network like this with a Cisco Switch with my home server, POE security system, mesh wifi access points, and ethernet ports accessible in most places in the house. I'll get to that when I get there though.

What I worked through

  • Physical path from modem to router to switch to mesh
  • Network separation for personal, household, and guest use
  • Signal testing and connectivity checks across the space
  • Validation that devices stayed on the correct network

Home Media Server deployment and Remote Access Workflow

Windows, storage organization, and remote administration.

This project focused on centralizing storage, keeping files organized, and making the server easier to manage day to day through remote tools. I have used Teamviewer and Microsoft Remote Desktop to get most of that done. I need to look into better ways to do it though. Remote desktop works great if I'm home, but I have had a lot of issues with it out of the local network. Teamviewer works good too, but if you use it too much they want you to pay for it and I like free opensource options. I hope to expand this with the network upgrade when we move as well with better internet and helping store media for my family and friends while working on my personal IT projects at home.

What I learned

  • Why storage planning matters before growth becomes a problem
  • How remote administration fits into practical maintenance
  • Where redundancy and backup planning should improve next

Open Demo View Notes

Payroll Calculator Notes

This demo is based on my C++ payroll coursework. The original idea started with basic input, calculations, and output for hours worked and hourly pay.

This web version expands the idea by adding overtime, filing status, estimated tax deductions, and a step-by-step math breakdown.

  • Regular pay is calculated from hours up to 40.
  • Overtime is calculated at 1.5 times the hourly rate.
  • Taxes are estimated using simplified demo logic.
  • Net pay is calculated after deductions and estimated taxes.

This is a class and portfolio demo, not an official payroll or tax tool.

Computer Build Planning and Hardware Recommendations

Compatibility checking, budgeting, and explaining tradeoffs clearly.

I enjoy comparing parts and creating realistic recommendations that balance price, upgrade potential, and performance for the person actually using the machine. Everyone has different needs and budgets when getting their own computer. There are a lot of options out there and I have spent time getting to know the ins and outs of parts and what they do. It's facinating to me, but the general people just want to know what they need to run what they want. Is it powerful enough to run this game without issues?
